## Affected Pages/Routes

- 企微测试群（`WECOM_CHAT_ID`）— 唯一 V1 验收面；无 Web UI

## Key Interactions to Verify

- 长连接：机器人安装到群后，成员发文本 → 服务日志/DB 有记录
- 手动总结：`bun run src/cli.ts trigger-summary` → 群内一条三段式总结
- 定时总结：工作日 18:00 自动出现总结（或 mock cron 触发）
- 空消息日：无文本时推送「今日无有效群消息」

## Edge Cases

- 重复 `msg_id` 不重复入库
- 17:50 发送、18:05 入库的消息仍算「当日」（msg_ts）
- LLM 失败后 CLI 重跑同日不双发（wecom_msg_id）
- 断连后 2 分钟内恢复收消息

## Critical Paths

1. Spike：openws 连接 → 收文本 → 发文本回群  
2. E2E：收一天消息 → trigger-summary → 群内总结可核对时间+发送人  
3. Eval：fixture 消息 → summarizer 输出含三段标题、无 DB 外人员
